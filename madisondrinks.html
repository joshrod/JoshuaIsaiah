MadisonDrinks

Mobile Development / UI Design

While trying to find my way in Madison after university, a group of friends and I that were still in town would get together at pubs and bars downtown on some days after work. We didn't have funds to carelessly spend, so we would seek out the cheapest places depending on the day we were available. This is where the idea for an application that would inform the user of the places where they could save money that day started.

Research and Development

With prior programming knowledge in Java from university, I began to learn how the Android Operating System worked at a deeper level. I learned about how activities, fragments, views, and Android XML files worked together to provide a functional application. The entire project was a challenge. Applying Java in different ways than I was accustomed to fried my brain many times.

I decided to focus on functionality rather than design for this project. What's the point of something looking great if its main purpose is to work properly?

I envisioned an application that was easy and intuitive to use. I wanted it to open up to the current day and display a list of the bars in downtown Madison. Tapping on the bar's name revealed its current drink specials as well as have the ability to open Google Maps to show the bar's location. With one swipe left or right, the day would change depending on the direction.

I used a Title Strip of the days of the week up at the top that would switch fragments in the activity when the user swiped the screen or touched the strip. The list of bars itself is an Expandable List View, comprised of a HashMap that used Strings as "keys", and a List of Strings as its "value."

On The Shelf...For Now

This project is currently an excellent starting point. I know it could look a lot better, but the important thing to note is that it works as intended. Eventually, I want to go back, optimize the code, and fix the application design. I can even see myself reworking this app to fit other cities as I continue learning.

My aspirations, however, cannot be fulfilled at the moment. During the application's development, I used my significant other's laptop rather than my current 2008 Macbook, and I didn't want to keep taking it while she was working on graduating university herself. While I was using the Android API for Eclipse, I found it much easier to use Android Studio, a program that my old Macbook cannot support.

The code for the application can be found on my GitHub profile, and the application itself can be found on the Google Play Store for your viewing pleasure.
